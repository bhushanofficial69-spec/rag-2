TASKS:

1. **Initialize Next.js 14 Project**
   - Create new Next.js 14 app with TypeScript enabled
   - Use App Router (NOT Pages Router)
   - Enable strict TypeScript mode (tsconfig.json strict: true)
   - Project name: codemind-rag

2. **Directory Structure**
   Create the following structure:
codemind-rag/
├── app/
│   ├── layout.tsx
│   ├── page.tsx
│   └── globals.css
├── components/
│   ├── ChatInterface.tsx
│   ├── CodeViewer.tsx
│   ├── FileTree.tsx
│   ├── RepoInput.tsx
│   └── ui/
│       └── Button.tsx (shadcn/ui)
├── lib/
│   ├── utils.ts
│   ├── api.ts
│   └── constants.ts
├── public/
├── styles/
│   └── globals.css
├── types/
│   └── index.ts
├── .env.example
├── .env.local (NOT in git)
├── tailwind.config.ts
├── tsconfig.json
├── next.config.js
├── package.json
├── PROJ_PLAN.md (CREATE THIS)
└── README.md

3. **Install Dependencies**
   Frontend dependencies:
   - react@latest
   - next@14
   - typescript@latest
   - tailwindcss@latest
   - postcss@latest
   - autoprefixer@latest
   - @monaco-editor/react@latest
   - prismjs@latest
   - axios@latest (for API calls)
   - clsx@latest (for conditional classes)
   - @radix-ui/react-slot@latest (for shadcn)
   - class-variance-authority@latest (for shadcn)

4. **Configure Tailwind CSS**
   - Set up tailwind.config.ts with proper theme configuration
   - Configure PostCSS (postcss.config.js)
   - Add Tailwind directives to globals.css

5. **Configure TypeScript**
   - Set tsconfig.json to strict mode
   - Add path aliases (@/ for src imports)
   - Enable strict null checks, strict function types

6. **Create .env.example**
```env
   # Backend API
   NEXT_PUBLIC_BACKEND_URL=http://localhost:8000
   NEXT_PUBLIC_API_TIMEOUT=30000

   # Feature Flags
   NEXT_PUBLIC_ENABLE_ANALYTICS=false
   NEXT_PUBLIC_MAX_QUERY_LENGTH=500
```

7. **Create PROJ_PLAN.md**
   This file will be the source of truth for the project state.
   Include:
   - Project overview (1 sentence)
   - Current phase number and name
   - Completed phases checklist
   - Current implementation status
   - Known issues (if any)
   - Next phase preview
   - Tech stack confirmed
   - API contract (to be updated each phase)
   - Database schema (to be updated)
   - Timestamps for phase completion

8. **Home Page Setup (app/page.tsx)**
   - Create a minimal home page that says "CodeMind RAG - Phase 1 Setup Complete"
   - Add a health check indicator
   - Show backend connection status (will be gray initially)
   - Create navigation placeholder for future components

9. **README.md**
   - Project description
   - Tech stack
   - Setup instructions
   - Development server instructions (npm run dev)
   - Phases overview
   - Current phase status

10. **Git Setup**
    - Initialize git (if not already done)
    - Create .gitignore (node_modules, .env.local, .next, dist, etc.)
    - Make initial commit: "Phase 1: Next.js 14 scaffolding complete"
