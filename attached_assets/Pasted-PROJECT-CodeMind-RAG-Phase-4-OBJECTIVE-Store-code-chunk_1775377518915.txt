3. **Create services/vector_db.py**
   Implement VectorDBClient class with:
   
   a) __init__(api_key: str, environment: str, index_name: str)
      - Initialize Pinecone client
      - Connect to index
      - Error handling: API key invalid, index not found
   
   b) Method: create_index_if_not_exists(index_name: str, dimension: int = 384)
      - Create Pinecone index if missing
      - Name: "codemind-codebase"
      - Dimension: 384 (matching embeddings)
      - Metric: "cosine"
      - Metadata filtering enabled
      - Logging: INFO "Index created" or "Index already exists"
   
   c) Method: upsert_chunk(chunk_id: str, vector: List[float], metadata: dict) -> bool
      - Upsert (insert or update) chunk into index
      - chunk_id: unique identifier (uuid)
      - vector: 384-dimensional embedding (placeholder for Phase 5)
      - metadata: CodeChunk fields as dict
      - Return: True if successful, False if error
      - Logging: DEBUG for each upsert
      - Error handling: Network error, invalid vector dimension
   
   d) Method: upsert_chunks_batch(chunks: List[Tuple[str, List[float], dict]]) -> int
      - Batch upsert multiple chunks (more efficient)
      - Return: count of chunks upserted
      - Logging: INFO "Upserted 1500 chunks"
   
   e) Method: query(vector: List[float], top_k: int = 10, filters: dict = None) -> List[dict]
      - Query similar vectors
      - vector: 384-dim query embedding
      - top_k: return top 10 results
      - filters: metadata filters (e.g., language="python")
      - Return: List of dicts with id, score, metadata
      - Will be used in Phase 6 (hybrid search)
   
   f) Method: delete_index(index_name: str) -> bool
      - Delete index (for cleanup/testing)
   
   g) Method: get_index_stats() -> dict
      - Return: index size, vector count, metadata
      - Logging: INFO with stats
